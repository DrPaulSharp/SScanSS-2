    sscanss.core.math.transform.find_3d_correspondence(source, query)¶

    Find Correspondence between 2 sets of 3D points by comparing pairwise distances. This method fails when points cannot be discriminated by their distances e.g in an equilateral triangle. source must not have less points than query and a minimum of 2 points is required to get correct results.

    Parameters:
    • source (numpy.ndarray) – array of 3D points,
    • query (numpy.ndarray) – array of 3D point.
    Returns:

    indices of correspondences
    Return type:

    numpy.ndarray

    sscanss.core.math.transform.rigid_transform(points_a, points_b)¶

    Calculate rigid transformation matrix given two sets of points

    S. Umeyama, Least-squares estimation of transformation parameters between two point patterns, IEEE Trans. Pattern Anal. Mach. Intell. 13 (4) (1991) 376- 380

    points_a must have the same number of points as points_b. A minimum of 3 points is required to get correct results

    Parameters:
    • points_a (numpy.ndarray) – array of 3D points.
    • points_b (numpy.ndarray) – array of 3D points.
    Returns:

    transformation matrix and residual errors
    Return type:

    TransformResult
    sscanss.core.math.transform.rotation_btw_vectors(v1, v2)¶

    Creates a rotation matrices to rotate from one vector (v1) to another (v2). Both vectors are assumed to be normalized. The implementation is based on Möller, Tomas, and John F. Hughes. “Efficiently building a matrix to rotate one vector to another.” Journal of graphics tools 4.4 (1999): 1-4.

    Parameters:
    • v1 (Vector3) – from unit vector
    • v2 (Vector3) – to unit vector
    Returns:

    rotation matrix
    Return type:

    Matrix33
